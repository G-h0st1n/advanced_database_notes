- Tables

    - Main column (PRIMARY KEY) is the ID
    - If referencing another table to avoid duplication you use a FK (Foreign Key). NOT necesarily the FK has to be the PK of the table its referencing
- Joins and inner Joins
    - Using a different bullet style
    - Even deeper sub-point
- Left Joins and Right Joins
    - a more specific way to find joins that go from the common things in Table a -> B (left) or from B -> A (Right)
- Cross Join
    - Each element in table a you combine with each element of table b
